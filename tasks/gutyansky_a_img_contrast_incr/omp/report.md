## 1. Контекст

OpenMP версия использует директивы компилятора для распараллеливания алгоритмов находждения минимума и максимума во входных данных, а также для преобразования данных.

OpenMP дает возможность простого добавления параллелизма в циклы.

## 3. Базовый алгоритм

Алгоритм идентичен SEQ, но два основных цикла (поиск мин./макс. элементов и преобразование) распараллелены.

## 4. Схема распараллеливания

**Первый цикл (поиск min/max):**

- Директива `#pragma omp parallel for` с `reduction(min:lower_bound, max:upper_bound)`.
- Переменные `input` и `sz` – `shared`, т.к. читаются всеми потоками
- синхронизация потоков происходит в конце параллельного цикла.

**Второй цикл (преобразование или копирование):**

- Если `delta != 0` – `#pragma omp parallel for` с `shared(input, output, sz, lower_bound, delta)` - гонки данных отсутствуют, т.к. все итерации цикла пишут в разные элементы выходного массива.
- Если `delta == 0` – аналогичная директива для копирования.
- `schedule` по умолчанию `static` - это дает лучший результат при равномерной вычислительной загруженности итераций.

## 5. Детали реализации

Файлы: `omp/include/ops_omp.hpp`, `omp/src/ops_omp.cpp`

Ключевые фрагменты:

```cpp
// Поиск минимума и максимума во входном массиве
#pragma omp parallel for default(none) shared(input, sz) reduction(min : lower_bound) reduction(max : upper_bound)
for (size_t i = 0; i < sz; i++) {
  uint8_t val = input[i];
  lower_bound = std::min(lower_bound, val);
  upper_bound = std::max(upper_bound, val);
}

// Преобразование элементов массива
#pragma omp parallel for default(none) shared(input, output, sz, lower_bound, delta)
for (size_t i = 0; i < sz; i++) {
  uint16_t old_value = input[i];
  uint16_t new_value = (255 * (old_value - lower_bound)) / delta;
  output[i] = static_cast<uint8_t>(new_value);
}
```

Риски гонок отсутствуют благодаря reduction и тому, что каждый поток пишет в разные элементы выходного массива.
